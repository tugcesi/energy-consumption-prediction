# Contributing to Energy Consumption Prediction

Thank you for your interest in contributing! Contributions are welcome and appreciated.

## How to Contribute

### Reporting Issues

- Use the [GitHub Issues](https://github.com/tugcesi/energy-consumption-prediction/issues) page to report bugs or suggest features.
- Please provide a clear description and steps to reproduce any bugs.

### Submitting Pull Requests

1. **Fork** the repository and create a new branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** and ensure the notebook runs without errors.

3. **Commit** your changes with a descriptive message:
   ```bash
   git commit -m "Add: brief description of your change"
   ```

4. **Push** to your fork and open a Pull Request against the `main` branch.

### Code Style

- Follow [PEP 8](https://peps.python.org/pep-0008/) guidelines for any Python code.
- Keep notebook cells clean and well-commented.
- Use descriptive variable names.

### Areas for Contribution

- Improving model accuracy (hyperparameter tuning, alternative architectures)
- Adding new visualizations or EDA steps
- Expanding documentation
- Adding unit tests for helper functions
- Supporting additional forecasting targets
